# はじめに

現在、スマホとPCで使える日常生活を快適にするアプリを開発しています。
ストップウォッチの機能・ボイスメモ機能・コーヒパッケージの画像認識語、テキストベース化・記録した時間をグラフ化などの機能を作成

# きっかけは？

作るきっかけとなったのは、今年の３月から研究室に配属され、9:30 ~ 17:00まで、研究室で過ごすことが増えたからです。
作業時間をはかったり、気づいたことをメモすることが増えたので、そのような日常使用する機能を自作して快適にしたいと考えたからです。
ＰＣとスマホで同期できるのを作りたいと考えたからです。

# つまづいた場所とどうやって解決したか

現在も制作している途中ですが、技術選定の部分で試行錯誤しました。
一つ目は、frontendをnext.jsで作成して、aws amplify でデプロイして、家のPCをサーバとして扱って開発を使用と試みた。利点として、URLから開けて、無料で使えて、セキュリティ面も悪くない。デメリットとして、家のPCを開いたままである、オフラインで使いにくい、フォーカスが個人に向きすぎていて、得た知見を情報発信むけではない。なので、途中まで作成して断念しました。
２つ目は、flutterとfirebaseを利用して、開発を行う。利点として、windowsで開発ができて、オフライン対応、複数端末対応ができる。デメリットとして、将来的に機能を増やしていった際にコストがかかりそう。あまり知見が得られなさそう。設計問題が多そう。なので、途中まで作成して、断念しました。

３つ目は、PWA方式を採用しました。

next.js + rails API + MYSQLを採用。

メリットとして、端末保存が簡単。オフラインで対応が簡単。機能が増やしやすい。知見が得やすい。
