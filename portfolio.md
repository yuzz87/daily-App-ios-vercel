# ポートフォリオ：日常生活支援アプリの開発

## 1. 自身で開発した成果物について

### 何を作ったか（概要）

スマホとPCの両方で使える、日常生活を快適にするWebアプリを開発しています。

主な機能：
- ストップウォッチ機能
- ボイスメモ機能
- コーヒーパッケージの画像認識によるテキスト化・記録
- 記録した時間のグラフ化

### 目的・背景（なぜ作ったか）

今年3月から研究室に配属され、9:30〜17:00まで研究室で過ごす時間が増えました。作業時間の計測や気づいたことのメモをする機会が増え、そのような日常的な機能を自作して快適にしたいと考えたことがきっかけです。また、PCとスマホで同期して使えるアプリを作りたいという目標もありました。

### 使用した技術

| 分類 | 技術 |
|------|------|
| フロントエンド | Next.js（PWA） |
| バックエンド | Ruby on Rails（API） |
| データベース | MySQL |
| デプロイ方式 | PWA（Progressive Web App） |

### 自分が担当した範囲

個人開発のため、設計・実装・インフラ構成まですべてを担当。

---

## 2. 技術的に一番悩んだ点と、その解決方法

### 何が問題だったか

最終的な技術スタックを決めるまでに、2つの構成を試みて断念しました。

### なぜそれが問題だったか・どう解決したか

**① Next.js ＋ AWS Amplify ＋ 自宅PCサーバ構成**

- メリット：URLからアクセス可能、無料運用、セキュリティ面も問題なし
- デメリット：
  - 自宅PCを常時起動しておく必要がある
  - オフラインで使いにくい
  - 個人利用に特化しすぎており、将来的な情報発信には向かない
- 結果：途中まで作成して断念

**② Flutter ＋ Firebase 構成**

- メリット：Windows上で開発可能、オフライン対応、複数端末対応
- デメリット：
  - 機能を増やしていくとコストが増大する懸念
  - 得られる知見が少ない
  - 設計上の問題が多く出てきた
- 結果：途中まで作成して断念

**③ PWA方式（Next.js ＋ Rails API ＋ MySQL）← 現在採用中**

上記2つの反省を踏まえ、PWA方式を採用することで以下の課題を解決しました。

- 端末へのローカル保存が簡単
- オフライン対応が容易
- 機能の追加・拡張がしやすい
- Rails / Next.js の知見が得やすく、技術力向上にも繋がる

現在もこの構成で開発を継続中です。
